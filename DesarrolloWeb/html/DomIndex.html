Preguntas

  1. ¿Qué es el DOM?

    2. El DOM (Document Object Model) es una representación en memoria de un documento HTML o XML que se carga en un navegador web. El DOM es una estructura de árbol que define cada elemento del documento como un objeto en el árbol, lo que permite a los desarrolladores web acceder y manipular los elementos y sus atributos mediante lenguajes de programación como JavaScript. Según Copilot.

  2. ¿Qué es un nodo?

    3. Un nodo es un objeto dentro del DOM que representa un elemento, atributo o texto en un documento HTML o XML. Según ChatGPT.

  3. ¿Cómo se accede a un elemento del DOM?

    4. Para acceder a un elemento del Document Object Model (DOM) en JavaScript, se puede utilizar el método document.getElementById() si el elemento que se desea acceder tiene un atributo de id definido en el HTML.Por ejemplo: si el HTML tiene un elemento '

    ' con un atributo id de "miDiv", se puede acceder a ese elemento en JavaScript de la siguiente manera: var elemento = document.getElementById("miDiv"); Según Copilot.

  4. ¿Cómo se modifica un elemento del DOM?

    5. Para modificar un elemento del DOM en JavaScript, se pueden utilizar varias propiedades y métodos del objeto DOM. Algunos ejemplos son: elemento.innerHTML para cambiar el contenido HTML de un elemento, elemento.style para cambiar los estilos CSS de un elemento, elemento.setAttribute() para establecer el valor de un atributo de un elemento y elemento.classList para agregar, eliminar o cambiar clases CSS en un elemento.

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    6. La diferencia entre getElementby* y querySelector* es que los métodos getElementby* obtienen elementos del DOM por su ID, etiqueta o clase, mientras que los métodos querySelector* permiten obtener elementos del DOM utilizando selectores de CSS. Según ChatGPT.

  6. ¿Qué es un evento?

    7. Es una acción que se produce en un elemento del DOM, como hacer clic en un botón, mover el ratón sobre un elemento, pulsar una tecla, etc. Según Copilot.

  7. ¿Qué es un listener?

    8. es una función que se utiliza para "escuchar" o detectar eventos en los elementos del DOM y realizar una acción en respuesta a ese evento. Para agregar un listener a un elemento del DOM, se puede utilizar el método addEventListener(). Según Copilot.

  8. ¿Qué es un callback?

    9. Es una función que se pasa como argumento a otra función y que se ejecuta cuando se completa una operación asincrónica. En el contexto de los listeners, un callback se utiliza para definir la acción que se realizará cuando se produzca un evento en un elemento del DOM. Según ChatGPT.

  9. ¿Qué es un objeto de evento?

    10. es un objeto que se crea automáticamente cuando se produce un evento en un elemento del DOM. El objeto de evento contiene información sobre el evento, como la posición del ratón, las teclas pulsadas, etc. Los objetos de evento se pasan como argumentos a las funciones de los listeners para que puedan acceder a esta información. Según Copilot.

Conclusiones

Experiencia general

Mi exprencia general con ambos es que tienen bastante que ofrecer y que ambos tienen sus ventajas y desventajas, son herramientas que si o si debemos de aprender a utilizar, y me refiero a aprender a usarlas no a depender de ellas.

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

En mi experiencia ChatGPT ya lo utilizaba desde hace tiempo y me parece una herramienta que me saca de apuros, en el único "problema" que observó es que para que ChatGPT te pueda dar una respuesta acertada y precisa necesita contexto acerca de lo que estas preguntando, y en cuento a Copilot a mi en lo personal no me funciono al 100% y tuvo que desinstalarlo e instalarlo varias veces cosa que me parece molesta, y siento que su fuerte es la programación pura, porque cuando le preguntas cosas que son de redactar rellena nada más la información o no contesta tan precisamente como Chat GPT.

Ventajas de usar AI para programar.

  • Reduce los tiempos que se utilizan para programar.
  • Puede identificar errores en el código rapidámente.
  • Puede sugerir una mejor optimización del código.
  • Nos puede explicar un código ajeno detalle a detalle.

    • Desventajas de usar AI para programar

  • Vuelve flojos a los programadores.
  • Puede volverse una dependencia.
  • No te puede dar una respuesta específica a tu problema, solo una sugerencia.
  • Muchhas veces mete código de relleno para poder completarlo.
